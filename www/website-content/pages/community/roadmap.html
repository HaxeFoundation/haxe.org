Haxe Roadmap

3.3 (November 2015)

Bytes support

Status:
Closed June 2015
Description

Use typed Arrays on JavaScript and emulate parts as necessary.

Finish static analyzer

Status:
Open since June 2015 (implementing)
Description

The static analyzer requires some more work to deal with edge cases and prepare for null-safety.

Unicode support

Status:
Open since May 2014 (planned)
Description

We want to add to the standard library, fully tested versions of the following classes:

  • haxe.Ucs2
  • haxe.Utf8
  • haxe.Utf16

They will be implemented as abstracts and will follow the String API + conversions between them and fromBytes/toBytes.

3.4 (May 2016)

Null-safety

Status:
Open since May 2013 (experimenting)
Description

Investigate in which ways the Haxe compiler can provide null-safety.

Rewrite PHP target

Status:
Open since June 2015 (experimenting)
Description

The current PHP target is difficult to maintain and targets an old version of PHP. We want to write a new generator which takes advantage of some newer PHP features

Make static analyzer the default

Status:
Open since June 2015 (planned)
Description

Once we can assume that it is stable the static analyzer should become the default compilation option, with a flag to disable it.
